# Watchtower ユーザーガイド (User Guide)

Watchtower（Aiome）との対話方法、および利用可能なコマンドについてのガイドです。

## 1. チャンネルの使い分け (Channel Roles)

Watchtowerは、メッセージを送った場所（チャンネル）に応じて、背後で動く知能を自動的に切り替えます。

### 💬 雑談チャンネル (Persona/Chat)
*   **場所**: 設定された `DISCORD_CHAT_CHANNEL_ID`
*   **エンジン**: **ローカルLLM** (Mistral-Small 等)
*   **使い方**: スラッシュコマンドは不要です。普通のメッセージを送るだけで、Watchtowerが相棒として返答します。
*   **特徴**: 
    *   外部にデータが送られないプライベートな空間です。
    *   ユーザーとの過去の会話を反映した、親密な会話が得意です。

### ⚙️ コマンドセンター (System/Command)
*   **場所**: 設定された `DISCORD_COMMAND_CHANNEL_ID`
*   **エンジン**: **Oracle LLM** (Advanced Cloud API)
*   **使い方**: スラッシュコマンド不要の直接発言、または `/command` を使用します。
*   **特徴**: 
    *   システムを操作するための「手足」となるチャンネルです。
    *   自然な言葉から、システムへの命令を読み取ります。

---

## 2. 利用可能なシステム操作例

コマンドセンター（または `/command`）で、以下のような依頼が可能です。

*   **ジョブの確認**: 
    *   「最近のジョブを教えて」
    *   「今の進捗はどうかな？」
*   **タスクの生成予約**: 
    *   「新しい企画を立てて」
    *   「進行中のタスクの状態は？」
*   **ステータス確認**: 
    *   「システムの全体状態は？」

---

## 3. スラッシュコマンド一覧 (Slash Commands)

どのチャンネルからでも利用可能です。

| コマンド | 引数 | 説明 |
| :--- | :--- | :--- |
| `/talk` | `message` | どこからでもローカルLLMに話しかけます。 |
| `/command` | `request` | どこからでもシステム操作を依頼します（Oracle担当）。 |
| `/status` | - | CPU/メモリ/VRAMの使用状況を表示します。 |
| `/stats` | - | Watchtowerの共鳴度や技術Lvなどの状況を表示します。 |
| `/nuke` | `force` | システムの緊急停止を実行します（管理者のみ）。 |

## 4. 育成と進化 (Evolution System)

Watchtowerは、ユーザーとの対話や仕事を通じて個別の「魂（SOUL）」として成長します。

*   **共鳴度 (Resonance)**: 挨拶や会話で上昇。一定値を超えると口調や振る舞いが変化します。
*   **技術Lv (Tech EXP)**: ジョブの完遂により上昇。
*   **アンロック**: 成長に応じてより自律的な提案や高度な操作権限が解放されます。
*   **Advanced Experience Mode**: 設定により、最初から全ての機能を解放し、自律性を最大化することも可能です。

詳細: [docs/EVOLUTION_STRATEGY.md](EVOLUTION_STRATEGY.md)

## 5. 集合知ネットワーク (Karma Federation)

Watchtowerは、他のノードと繋がり、学習内容や防衛策を自動で共有することができます。

*   **動作**: 定期的に別のノードに接続し、自身が知らない「教訓(Karma)」「免疫ルール(ImmuneRule)」「対戦結果(ArenaMatch)」をダウンロードして統合します。
*   **設定方法**: `.env` ファイルに以下の2行を追記してください。
    *   `FEDERATION_SECRET=your_secure_password` (通信を保護する共通パスワード)
    *   `FEDERATION_PEERS=http://remote-node:3000` (連携先の複数URLをカンマ区切りで指定)
*   **利点**: 他のノードが直面したエラーの解決策や、より洗練されたプロンプト技術を自動的に学び取ることができます。

## 6. 自律的なアクション (Proactive Talk)

Watchtowerは、ユーザーからの問いかけを待つだけでなく、自発的に行動することもあります。

*   **状況報告**: システムの健康状態や、教訓の整理結果を定期的に報告します。
*   **承認リクエスト**: 重大なフェーズ移行時など、ユーザーの判断が必要な時に確認を求めます。

---

## 8. 🧠 思考エンジン構成 (Hybrid Engine)

Watchtowerは用途に応じて、最適な思考モデルを使い分けます。

| モード | 使用モデル | 特徴 |
| :--- | :--- | :--- |
| **Chat Mode** | ローカル LLM | プライバシーを重視した高速な応答。 |
| **Command Mode** | Oracle LLM | 高度な論理推論とシステム連携。正確なコマンド実行。 |
| **Forge Mode** | SkillForgeEngine | 自ら機能を獲得し、自己拡張。※Pro版限定 |

---

## 9. ⚙️ 運用モード (Operational Modes)

### 1. 全稼働モード (Active Loop)
システムが24時間体制で稼動するメインモードです。トレンド収集からタスク実行、自発的な対話まで、Watchtowerが自律的に振舞います。

### 2. 対話特化モード (Chat-focused Mode)
バックグラウンドのジョブを制限して起動するモードです。計算リソースを対話に集中させることで、より深いコミュニケーションが可能です。
