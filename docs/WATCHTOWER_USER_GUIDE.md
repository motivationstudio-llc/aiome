# Watchtower ユーザーガイド (User Guide)

Watchtower（Aiome）との対話方法、および利用可能なコマンドについてのガイドです。

## 1. チャンネルの使い分け (Channel Roles)

Watchtowerは、メッセージを送った場所（チャンネル）に応じて、背後で動く知能を自動的に切り替えます。

### 💬 雑談チャンネル (Persona/Chat)
*   **場所**: 設定された `DISCORD_CHAT_CHANNEL_ID`
*   **エンジン**: **ローカルLLM** (Mistral-Small 等)
*   **使い方**: スラッシュコマンドは不要です。普通のメッセージを送るだけで、Watchtowerが相棒として返答します。
*   **特徴**: 
    *   外部にデータが送られないプライベートな空間です。
    *   マスターとの過去の会話を反映した、親密な会話が得意です。

### ⚙️ コマンドセンター (System/Command)
*   **場所**: 設定された `DISCORD_COMMAND_CHANNEL_ID`
*   **エンジン**: **Gemini 2.0 Flash** (Cloud API)
*   **使い方**: スラッシュコマンド不要の直接発言、または `/command` を使用します。
*   **特徴**: 
    *   システムを操作するための「手足」となるチャンネルです。
    *   人間のような柔軟な言葉から、システムへの命令を読み取ります。

---

## 2. 利用可能なシステム操作例

コマンドセンター（または `/command`）で、以下のような依頼が可能です。

*   **ジョブの確認**: 
    *   「最近のジョブを教えて」
    *   「今の進捗はどうかな？」
*   **動画の生成予約**: 
    *   「『AIの未来』についてテック系の動画を作って」
    *   「新しく動画の企画を立てて」
*   **ステータス確認**: 
    *   「ファクトリーの状態は？」

---

## 3. スラッシュコマンド一覧 (Slash Commands)

どのチャンネルからでも利用可能です。

| コマンド | 引数 | 説明 |
| :--- | :--- | :--- |
| `/talk` | `message` | どこからでもローカルLLMに話しかけます。 |
| `/command` | `request` | どこからでもシステム操作を依頼します（Gemini担当）。 |
| `/status` | - | CPU/メモリ/VRAMの使用状況を表示します。 |
| `/stats` | - | Watchtowerの親愛度や技術Lvなどの育成状況を表示します。 |
| `/nuke` | `force` | システムの緊急停止を実行します（管理者のみ）。 |

---

## 4. 育成と進化 (Evolution System)

Watchtowerは、マスターとの日々の交流や仕事を通じて成長します。

*   **親愛度 (Affection)**: 挨拶や会話で上昇。一定値を超えると口調が変化します。
*   **技術Lv (Tech EXP)**: 動画生成ジョブが成功すると上昇。
*   **アンロック**: 成長に応じて「タメ口モード」や「Deep Night Mode（親密・R18）」などが順次解放されます。
*   **Unleashed Mode**: プロダクト設定により、最初から全ての機能をフルオープンにすることも可能です。

詳細: [docs/EVOLUTION_STRATEGY.md](EVOLUTION_STRATEGY.md)

## 5. 集合知ネットワーク (Karma Federation)

Watchtowerは、他のノード（別のPCやサーバーで動くWatchtower）と繋がり、学習内容や防衛策を自動で共有することができます。

*   **動作**: 約5分おきに指定したピア（他ノード）に接続し、自身が知らない「教訓(Karma)」「免疫ルール(ImmuneRule)」「対戦結果(ArenaMatch)」をダウンロードして統合します。
*   **設定方法**: `.env` ファイルに以下の2行を追記してください。
    *   `FEDERATION_SECRET=your_secure_password` (通信を保護する共通パスワード)
    *   `FEDERATION_PEERS=http://another-pc:3000` (連携先の複数URLをカンマ区切りで指定)
*   **利点**: 自分がまだ直面していないエラーの解決策や、より洗練されたプロンプト技術を、他のWatchtowerから自動的に学び取ることができます。

## 6. 自律的なアクション (Proactive Talk)

Watchtowerは、マスターからの問いかけを待つだけでなく、自分から話しかけることもあります。

*   **朝の挨拶 (09:00)**: その日の調子や挨拶を送ります。
*   **記憶の整理報告 (深夜)**: 昨日の会話を振り返り、自分の中で整理できたことを報告します。
*   **承認リクエスト**: 動画生成のフェーズ移行時など、マスターの判断が必要な時にボタン付きメッセージを送ります。

## 8. 🧠 思考エンジン構成 (Hybrid Engine)

Watchtowerは用途に応じて、最適な思考モデルを使い分けます。

| モード | 使用モデル | 特徴 |
| :--- | :--- | :--- |
| **Chat Mode** | ローカル LLM | 人格とプライバシーを重視した高速な応答。 |
| **Command Mode** | Gemini 2.0 Flash | 高度な論理推論とシステム連携。正確なコマンド実行。 |
| **Forge Mode** | WASM SkillForge | 自らコードを生成し、技能として獲得。※Pro版限定 |

## 9. ⚙️ 運用モード (Operational Modes)

### 1. Samsara 全稼働モード (Active Loop)
システムが24時間体制で稼働するメインモードです。トレンド収集から動画生成、マスターへの自発的な話しかけまで、Watchtowerが「生きている」ように振る舞います。

### 2. 対話特化モード (Chat-only Mode)
動画生成（Cron）をオフにして起動するモードです。生成リソースを全て対話に振り向けるため、より巨大なモデルでの雑談や深いメンタルケアに適しています。
