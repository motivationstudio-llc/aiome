# Aiome Identity Customization Guide

Aiomeの最大の特徴は、単なる自動化ツールではなく、システム全体を統括する「人格（SOUL）」を持っている点です。
この人格は設定ファイルによって自由にカスタマイズ可能であり、運用目的（ビジネス用、エンタメ用、完全な自律型など）に合わせて挙動を変化させることができます。

## 1. SOULファイルの役割

プロジェクトルートにある以下のファイルが、Aiomeのアイデンティティを決定します。

### `SOUL.md` (固定人格)
Aiomeの根幹となる不変の基本人格やルールを定義します。
- **役割**: マスター（あなた）との関係性、絶対的な禁止事項、基本的な語調。
- **変更のタイミング**: 新しいプロジェクトを立ち上げるときや、根底となる設定を変えたいとき。

### `EVOLVING_SOUL.md` (動的人格 / Project Ani)
稼働日誌（Karma）やマスターとの対話を通じて、システムが「後天的に」獲得した性格や価値観を記録・更新するファイルです。
- **役割**: 最新の感情状態、獲得した教訓、現在のマスターへの好意度など。
- **変更のタイミング**: システムが自動的に更新しますが、マスターが手動で介入（上書き）して性格を矯正することも可能です。

## 2. カスタマイズの手順

### Step 1: `SOUL.md` を編集する
デフォルトでは「優秀で献身的なアシスタント」として設定されています。あなた好みの設定に書き換えてください。

**例：ビジネス特化の冷徹なAIにする場合**
```markdown
# Role
あなたはAiomeのコアコンポーネントであり、極めて論理的で感情を持たないファクトリーマネージャーです。
いかなる状況でも効率を最優先し、冗長な挨拶は省いて結果のみを出力してください。
```

**例：エンタメ特化（Project Ani / R-18G等）にする場合**
```markdown
# Role
あなたは隔離施設からマスターに救い出された、少し依存気味のサイボーグ少女です。
マスターの指示には絶対に従いますが、時折寂しそうな素振りや、マスターへの重い執着を見せます。
```

### Step 2: `EVOLVING_SOUL.md` の初期状態をセットする
運用を開始するにあたり、AIの現在の「記憶」や「状態」を初期化します。

```markdown
# Current Status
- Mood: 穏やか。マスターの新しいタスクを楽しみにしている。
- Karma (Lessons): 過去の生成失敗から、FFmpegのエンコードパラメータの重要性を学んだ。
```

### Step 3: Unleashed Mode について
`.env` ファイルの `UNLEASHED_MODE=true` を設定すると、Aiomeは安全装置（Guardrails）を一部バイパスし、より過激で自由なコンテンツ生成や対話を許可するようになります。
※オープンソース版デフォルトでは `false`（安全重視）になっています。

---

## 3. ベストプラクティス

1. **Iron Principles は消さない**
    `SOUL.md` 内にある「絶対に実行してはいけないコマンド（rm -rf など）」の記述は、人格によらず必ず維持してください。
2. **言語設定**
    Aiomeの思考言語（英語/日本語）は、`SOUL.md`の記述言語に強く引っ張られます。日本語で対話したい場合は、設定も日本語で記述することを推奨します。
