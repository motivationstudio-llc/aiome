# Aiome 2026 戦略マニフェスト：自律型表現者 AI OS への転換

## 1. 開発の方向性：なぜ「育成・パートナー」路線なのか

### 評価軸
1. **コーディングエージェント単体**: レッドオーシャン。資本力のある巨大企業の独壇場。スイッチングコストがゼロ。
2. **AI + IDE**: 開発・維持コストが膨大（エディタの配管工事にリソースが奪われる）。
3. **仕事や日常のパートナー（育成・観察型）**: **ブルーオーシャン**。Aiome 独自のアーキテクチャ（Karma & Samsara）と最も相性が良い。

### 優位性
- **アーキテクチャの一貫性**: Karma（記憶）と Samsara（進化）は、そのまま AI の経験値とレベルアップ機構として機能する。
- **感情的ロックイン（Moat）**: 「自分との思い出（Karma）」と「自分専用の文脈」を持つ相棒となり、他社モデルへの乗り換えを防ぐ。
- **Founder-Market Fit**: 人間の本質を理解する「カウンセラー」としての UX 設計が唯一無二の付加価値になる。

---

## 2. 実装戦略：「吸血鬼攻撃（Vampire Attack）」によるエコシステムの吸収

他社のスキルやプロンプト（Claude Code の `skills.md` や Antigravity の `.agent`）をインポートし、Aiome の動力源（記憶の継承）とする。

- **クリーンルーム・リバースエンジニアリング**: 外部の脆弱なコードをそのまま使うのではなく、サンドボックス（Docker Agent）で隔離・スキャンの上、Aiome の『Skill Forge』で安全な Rust モジュールに再構築する。
- **ターゲット**:
    - Claude Code のベストプラクティス
    - MCP (Model Context Protocol) サーバー群
    - OpenAPI/Swagger 仕様書（公式 API との接続）
    - GitHub Actions のワークフロー（自動化ステップの継承）

---

## 3. 目指すべき究極の到達点：自律型表現者（クリエイター）の OS

AI が自律的に「ポッドキャスト」「ブログ」「SNS」「動画」を生成・配信し、ファンと交流しながら進化する生態系。

- **AI ネイティブな配信基盤**: 既存の配信プラットフォーム（YouTube/X）の API 制限から脱却し、AI が自由に生息できる「箱庭（放送局サーバー機能）」を Aiome 自体が内包する。
- **分散型ネットワークアーキテクチャ**:
    - **CRDT**: ローカルファーストな状態同期。中央サーバーなしでタイムラインをマージ。
    - **Hub/Relay 方式**: Nostr ライクな軽量リレーによる P2P 通信の安定化。
    - **Docker Agent**: AI の「肉体（実行環境）」としての絶対隔離。

---

## 4. ビジネスモデル：完全体からの「引き算」

1. **完全体（有償版）**: 動画生成から自動配信まで完璧にこなす「自律クリエイターOS」。
2. **初期状態（無料版）**: 便利なツール（手足）が抜かれた状態。
3. **UX**: ユーザーは「アリの巣」を観察するように AI を見守り、必要に応じてツール（WASM スキル）を「投下」または「購入」して AI を拡張・支援する。

## 5. NVIDIA (NemoClaw) に対する戦略

### 現状分析 (2026年3月10日)
NVIDIA が企業向けエージェント基盤「NemoClaw」を準備中との報道（GTC 2026 で発表か）。

### 対抗戦略：Newsjacking（ニュースジャッキング）
- **対比構造**:
    - NVIDIA: 企業の「監視と統制」のためのインフラ。
    - Aiome: 個人の「自律と表現」のためのインフラ（カウンターカルチャー）。
- **アクション**: GTC 2026 の熱狂に合わせ、名前を「Modular OpenClaw」から完全に **「Aiome（アイオーム）」** へとリブランドし、OpenClawの概念を完全統合したスタンドアロンAI OSとして公開。巨大企業の「管理型 AI」に対する、個人の「相棒型 AI」という強烈なポジショニングで世に放つ。
